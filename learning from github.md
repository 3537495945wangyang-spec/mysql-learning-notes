# 检索增强生成（RAG）与联网搜索的区别
### 核心共同点

1. **基本思路一致**：两者都遵循“**检索 + 生成**”的模式。首先从外部知识库（无论是内部文档还是互联网）中查找相关信息，然后将其作为上下文提供给LLM，最终生成融合了这些外部信息的回答。
    
2. **解决LLM的固有问题**：都旨在弥补LLM的“知识冻结”（不知道新信息）和“幻觉”（编造信息）的缺陷。
    

### 本质区别

关键在于 **“知识源的性质”**​ 和 **“系统的开放性”**。

- **RAG**​ 更像是为企业或特定领域**建造一个私有的、定制化的“知识增强大脑”**。你需要自己准备“食粮”（私有数据），自己设计“消化系统”（检索流程），最终得到一个专门为你服务的智能体。
    
- **联网搜索**​ 更像是为通用的AI助手**配备了一个通往公共图书馆（互联网）的标准化借阅卡**。它用固定的方式获取公共信息来补充回答，但你不能改变图书馆的藏书（知识源），也不能定制借阅流程。
    

**结论**：虽然底层思想同源，但RAG是一个更底层、更灵活、面向企业和开发者的**技术框架**，用于构建基于私有知识的智能应用；而AI软件的联网搜索是一个面向最终用户的、功能固定的**产品特性**，用于获取实时公共信息。前者提供了高度的定制化和控制力，后者则提供了开箱即用的便利性。
